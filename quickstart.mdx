This guide will help you set up and run the CoW Protocol Backend for Frontend (BFF) locally.

## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js (v18 or higher)
- Yarn package manager
- Docker and Docker Compose
- Git

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the BFF repository to your local machine:
    
    ```bash
    git clone https://github.com/cowprotocol/bff.git
    cd bff
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required Node.js dependencies using Yarn:
    
    ```bash
    yarn
    ```
    
    This will also automatically generate API types from Swagger definitions via the `postinstall` hook.
  </Step>

  <Step title="Configure environment variables">
    Create your environment configuration file from the example template:
    
    <CodeGroup>
    ```bash Local Development
    cp .env.example .env
    ```
    
    ```bash Docker Development
    cp .env.example .env.docker
    ```
    </CodeGroup>
    
    <Tip>
    Use `.env` for running services locally via NX commands, and `.env.docker` when running services through Docker Compose.
    </Tip>
    
    Key environment variables to configure:
    
    ```bash .env
    # Port for the API service (default: 3001)
    PORT=3001
    
    # RPC URLs for blockchain networks
    RPC_URL_1=https://mainnet.infura.io/v3/your-infura-key
    RPC_URL_100=https://rpc.gnosischain.com
    
    # CoW Protocol API endpoints
    COW_API_BASE_URL=https://api.cow.fi
    COW_API_URL_1=https://api.cow.fi/mainnet
    COW_API_URL_100=https://api.cow.fi/xdai
    
    # Database configuration
    DATABASE_ENABLED=true
    DATABASE_HOST=localhost
    DATABASE_USERNAME=bff-db-user
    DATABASE_PASSWORD=bff-db-password
    DATABASE_NAME=bff-db
    DATABASE_PORT=5432
    
    # Redis cache (optional but recommended)
    REDIS_ENABLED=true
    REDIS_HOST=localhost
    REDIS_PORT=6379
    REDIS_USER=cow_redis
    REDIS_PASSWORD=cow_password
    
    # RabbitMQ for notifications
    QUEUE_USER=rabbit
    QUEUE_PASSWORD=my-rabbit-password
    QUEUE_HOST=localhost
    QUEUE_PORT=5672
    ```
  </Step>

  <Step title="Start infrastructure services">
    Launch the required infrastructure services (PostgreSQL, RabbitMQ, Redis) using Docker Compose:
    
    ```bash
    yarn compose:up
    ```
    
    This command will:
    1. Build Docker images for affected services
    2. Start PostgreSQL database on port `5432`
    3. Start RabbitMQ message queue on port `5672` (management UI on `15672`)
    4. Start Redis cache on port `6379`
    
    <Info>
    To start only specific services, use Docker Compose directly:
    ```bash
    docker-compose up -d db redis queue
    ```
    </Info>
  </Step>

  <Step title="Run database migrations">
    Initialize the database schema by running migrations:
    
    ```bash
    yarn migration:run
    ```
    
    This creates the necessary tables for storing notification subscriptions, indexer state, and other data.
  </Step>

  <Step title="Start the API service">
    Launch the main API service:
    
    ```bash
    yarn start
    ```
    
    The API server will start on `http://localhost:3001` by default.
    
    <Tip>
    You can also use `yarn start:api` which is an alias for the same command.
    </Tip>
  </Step>
</Steps>

## Verify Installation

Once the API is running, verify everything is working correctly:

<Steps>
  <Step title="Check the about endpoint">
    Test the basic API health:
    
    ```bash
    curl http://localhost:3001/about
    ```
    
    Expected response:
    ```json
    {
      "name": "BFF API",
      "version": "0.24.0",
      "gitCommitHash": "..."
    }
    ```
  </Step>

  <Step title="Test a real API endpoint">
    Fetch token balances for an Ethereum address:
    
    ```bash
    curl -X GET "http://localhost:3001/1/accounts/0x87A04752E516548B0d5d4DF97384C0b22B649179/balances?tokens=0x4d5f47fa6a74757f35c14fd3a6ef8e3c9bc514e8,0x6b175474e89094c44da98b954eedeac495271d0f"
    ```
    
    This fetches DAI and another token balance for the specified address on Ethereum mainnet (chainId: 1).
  </Step>

  <Step title="Access the Swagger documentation">
    Open your browser and navigate to:
    
    ```
    http://localhost:3001/docs
    ```
    
    This provides an interactive API documentation interface powered by Swagger UI.
  </Step>
</Steps>

## Running Additional Services

The BFF includes several other services beyond the main API:

### Notification Producer

The notification producer monitors blockchain events and generates push notifications:

```bash
# Start for all supported chains
yarn producer

# Start for specific chains only (mainnet and Gnosis)
NOTIFICATIONS_PRODUCER_CHAINS=1,100 yarn producer
```

### Telegram Bot

The Telegram notification consumer requires a bot token:

```bash
# Set your Telegram bot secret in .env
TELEGRAM_SECRET=your-bot-token-here

# Start the Telegram bot
yarn telegram
```

### TWAP Service

The Time-Weighted Average Price service:

```bash
# Run TWAP migrations first
yarn twap:run-migrations

# Start TWAP service (via Docker)
docker-compose up twap
```

## Development Workflow

### Running Tests

```bash
# Run all tests
yarn test

# Run tests for a specific library in watch mode
nx run repositories:test --watch

# Run a specific test file
nx run repositories:test --watch --testFile=libs/repositories/src/UsdRepository/UsdRepositoryRedis.spec.ts
```

### Linting and Type Checking

```bash
# Run linter for all projects
yarn lint

# Build all projects to check for type errors
yarn build
```

### Database Operations

<CodeGroup>
```bash Create Migration
# Create a new empty migration
yarn typeorm migration:create src/migrations/your-migration-name

# Or generate from entity changes
yarn migration:generate
```

```bash Run Migrations
# Apply pending migrations
yarn migration:run
```

```bash Revert Migration
# Rollback the last migration
yarn migration:revert
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="DriverPackageNotInstalledError: Postgres package not found">
    If you encounter this TypeORM error:
    
    ```
    DriverPackageNotInstalledError: Postgres package has not been found installed. Try to install it: npm install pg --save
    ```
    
    This is typically a TypeORM module resolution issue. The `pg` package is already installed as a dependency.
    
    **Solution**: Clear your node_modules and reinstall:
    ```bash
    rm -rf node_modules
    yarn install
    ```
    
    See [PR #101](https://github.com/cowprotocol/bff/pull/101) for more details.
  </Accordion>

  <Accordion title="Docker containers fail to start">
    If services fail to start via Docker Compose:
    
    1. Check that ports are not already in use:
       ```bash
       lsof -i :5432  # PostgreSQL
       lsof -i :5672  # RabbitMQ
       lsof -i :6379  # Redis
       ```
    
    2. Ensure Docker has enough resources allocated (especially on macOS)
    
    3. Try rebuilding images:
       ```bash
       docker-compose down -v
       yarn compose:up
       ```
  </Accordion>

  <Accordion title="API returns 500 errors for balance queries">
    Balance queries require:
    1. Valid RPC URLs configured for the target chain
    2. Redis running (for caching) or `REDIS_ENABLED=false`
    3. Proper network configuration
    
    Check the API logs for specific error messages:
    ```bash
    yarn start
    # Look for ERROR level logs
    ```
  </Accordion>
</AccordionGroup>
