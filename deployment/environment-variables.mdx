Configure the BFF services using environment variables. Copy `.env.example` to `.env.docker` for Docker deployments or `.env` for local development.

```bash
# For Docker Compose
cp .env.example .env.docker

# For local development
cp .env.example .env
```

## Application Configuration

### Version and Logging

<ParamField path="VERSION" type="string" default="1.0.0">
  Application version exposed in the `/about` endpoint. Useful for tracking Docker image versions.
</ParamField>

<ParamField path="LOG_FORMAT" type="string" default="pretty">
  Log output format. Options:
  - `pretty` - Human-readable format for development
  - `json` - Structured JSON format for production
</ParamField>

<ParamField path="LOG_LEVEL" type="string" default="info">
  Logging level. Options: `debug`, `info`, `warn`, `error`
</ParamField>

### Port Configuration

<ParamField path="PORT" type="number" default="8080">
  Port number for the API service.
  
  - API service default: `1500`
  - TWAP service default: `3000`
</ParamField>

## Database Configuration

<ParamField path="DATABASE_ENABLED" type="boolean" default="false">
  Enable PostgreSQL database. Set to `true` for services that require persistence (API, TWAP, notification producer).
</ParamField>

<ParamField path="DATABASE_HOST" type="string" default="localhost">
  PostgreSQL host. Use `db` for Docker Compose deployments.
  
  ```bash
  # Docker Compose
  DATABASE_HOST=db
  
  # Local development
  DATABASE_HOST=localhost
  ```
</ParamField>

<ParamField path="DATABASE_PORT" type="number" default="5432">
  PostgreSQL port
</ParamField>

<ParamField path="DATABASE_USERNAME" type="string" default="bff-db-user">
  PostgreSQL username
  
  <Warning>Use strong credentials in production</Warning>
</ParamField>

<ParamField path="DATABASE_PASSWORD" type="string" default="bff-db-password">
  PostgreSQL password
  
  <Warning>Never commit production passwords to version control</Warning>
</ParamField>

<ParamField path="DATABASE_NAME" type="string" default="bff-db">
  PostgreSQL database name

## Redis Cache Configuration

<ParamField path="REDIS_ENABLED" type="boolean" default="false">
  Enable Redis caching. Highly recommended for production to reduce RPC calls.
</ParamField>

<ParamField path="REDIS_HOST" type="string" default="localhost">
  Redis host. Use `redis` for Docker Compose deployments.
  
  ```bash
  # Docker Compose
  REDIS_HOST=redis
  
  # Local development
  REDIS_HOST=localhost
  ```
</ParamField>

<ParamField path="REDIS_PORT" type="number" default="6379">
  Redis port
</ParamField>

<ParamField path="REDIS_USER" type="string" default="cow_redis">
  Redis username (configured in `conf/redis/redis.conf`)
</ParamField>

<ParamField path="REDIS_PASSWORD" type="string" default="cow_password">
  Redis password
  
  <Warning>Change default password in production</Warning>
</ParamField>

<Accordion title="Redis Configuration Details">
  The Redis configuration in `conf/redis/redis.conf` includes:
  
  - Password authentication
  - Memory limit: 256MB
  - Eviction policy: LRU (Least Recently Used)
  - Disabled dangerous commands (FLUSHDB, FLUSHALL, CONFIG)
  - Persistence: Disabled by default for caching
</Accordion>

  RabbitMQ host. Use `queue` for Docker Compose deployments.

  ```bash
  QUEUE_HOST=queue
  ```

</ParamField>

<ParamField path="QUEUE_PORT" type="number" default="5672">
  RabbitMQ AMQP port
</ParamField>

<ParamField path="QUEUE_USER" type="string" default="rabbit">
  RabbitMQ username
</ParamField>

<ParamField path="QUEUE_PASSWORD" type="string" default="my-rabbit-password">
  RabbitMQ password
  
  <Warning>Change default password in production</Warning>
</ParamField>

<Info>
  RabbitMQ management UI is available at `http://localhost:15672` with the same credentials.
</Info>

<ParamField path="NOTIFICATIONS_PRODUCER_CHAINS" type="string">
  Comma-separated list of chain IDs to monitor for notifications.
  
  ```bash

  NOTIFICATIONS_PRODUCER_CHAINS=1,100
  
  # Monitor all supported networks (default if not set)
  NOTIFICATIONS_PRODUCER_CHAINS=1,56,100,137,8453,42161,43114
  ```
</ParamField>

## Service-Specific Variables

### API Service

The API service uses all common variables plus:

```bash
PORT=1500
PROXY_UPSTREAM=https://api.coingecko.com
PROXY_ORIGIN=https://cow.fi
PROXY_HOST=api.coingecko.com
JWT_CERT_PASSPHRASE=your-secret-passphrase
INFURA_KEY=your-infura-key
INFURA_JWT_ID=your-jwt-id
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_USER=cow_redis
REDIS_PASSWORD=cow_password
TWAP_BASE_URL=http://twap:3000/
```

### TWAP Service

The TWAP service requires:

```bash
PORT=3000
DATABASE_ENABLED=true
DATABASE_HOST=db
DATABASE_PORT=5432
DATABASE_USERNAME=bff-db-user
DATABASE_PASSWORD=bff-db-password
DATABASE_NAME=bff-db
```


## Validation

Validate your environment configuration:

```bash
# Check required variables are set
docker-compose config

# Test service connectivity
docker-compose up -d db redis queue
sleep 5
docker-compose exec api yarn test
```

## Security Best Practices

<Warning>
  **Never commit sensitive environment files to version control**
  
  Add to `.gitignore`:
  ```
  .env
  .env.docker
  .env.local
  .env.production
  ```
</Warning>

<CardGroup cols={2}>

  </Card>
  <Card title="Rotate Credentials" icon="rotate">
    Regular rotate API keys, database passwords, and JWT passphrases
  </Card>
  <Card title="Principle of Least Privilege" icon="shield">
