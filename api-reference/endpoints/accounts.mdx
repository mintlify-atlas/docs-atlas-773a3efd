Account endpoints provide real-time information about user token balances and allowances for the CoW Protocol.

## Get Account Balances

<CodeGroup>
```bash cURL
curl "https://api.cow.fi/1/accounts/0x1234567890abcdef1234567890abcdef12345678/balances?tokens=0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2,0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48"
```

```javascript JavaScript
const userAddress = '0x1234567890abcdef1234567890abcdef12345678';
const tokens = [
  '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // WETH
  '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48'  // USDC

const response = await fetch(
  `https://api.cow.fi/1/accounts/${userAddress}/balances?tokens=${tokens.join(',')}`
);
const balances = await response.json();
console.log(balances);
```

```python Python
import requests

user_address = '0x1234567890abcdef1234567890abcdef12345678'

    '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2',  # WETH
```

Retrieves token balances and allowances for a user address.

**Endpoint:** `GET /{chainId}/accounts/{userAddress}/balances`

### Path Parameters

<ParamField path="chainId" type="string" required>
  The blockchain network ID (e.g., `1`, `100`, `11155111`)
</ParamField>

<ParamField path="userAddress" type="string" required>
  The user's wallet address
</ParamField>

### Query Parameters

<ParamField query="tokens" type="string" required>
  Comma-separated list of token addresses to query
</ParamField>

### Response

Returns an array of token balance objects.

<ResponseField name="balance" type="string" required>
  User balance in token units (wei-like format)
</ResponseField>

<ResponseField name="allowance" type="string" required>
  Allowance for CoW Protocol vault relayer in token units
</ResponseField>

<ResponseField name="token" type="object" required>
  Token information object
</ResponseField>

<ResponseField name="token.address" type="string" required>
  Token contract address
</ResponseField>

<ResponseField name="token.decimals" type="number">
  Token decimals (e.g., 18 for most tokens)
</ResponseField>

<ResponseField name="token.symbol" type="string">
  Token symbol (e.g., "WETH", "USDC")
</ResponseField>

<ResponseField name="token.name" type="string">
  Token name (e.g., "Wrapped Ether")
</ResponseField>

## Stream Account Balances (SSE)

<CodeGroup>
```bash cURL
curl -N "https://api.cow.fi/1/accounts/0x1234567890abcdef1234567890abcdef12345678/balances/sse?tokens=0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2,0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48"
```

```javascript JavaScript
const userAddress = '0x1234567890abcdef1234567890abcdef12345678';
const tokens = [
  '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2',
  '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48'
];

const eventSource = new EventSource(
  `https://api.cow.fi/1/accounts/${userAddress}/balances/sse?tokens=${tokens.join(',')}`
);

eventSource.onmessage = (event) => {
  const balances = JSON.parse(event.data);
  console.log('Balance update:', balances);
};

eventSource.addEventListener('ping', (event) => {
  console.log('Keep-alive ping received');
});

eventSource.onerror = (error) => {
  console.error('SSE error:', error);
  eventSource.close();
};
```

```python Python
import requests
import json

user_address = '0x1234567890abcdef1234567890abcdef12345678'
tokens = [
    '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2',
    '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48'
]

url = f'https://api.cow.fi/1/accounts/{user_address}/balances/sse'
params = {'tokens': ','.join(tokens)}

with requests.get(url, params=params, stream=True) as response:
    for line in response.iter_lines():
        if line:
            decoded = line.decode('utf-8')
            if decoded.startswith('data: '):
                data = json.loads(decoded[6:])
                print('Balance update:', data)
```
</CodeGroup>

Streams real-time token balance and allowance updates using Server-Sent Events (SSE).

**Endpoint:** `GET /{chainId}/accounts/{userAddress}/balances/sse`

### Path Parameters

<ParamField path="chainId" type="string" required>
  The blockchain network ID (e.g., `1`, `100`, `11155111`)
</ParamField>

<ParamField path="userAddress" type="string" required>
  The user's wallet address
</ParamField>

### Query Parameters

<ParamField query="tokens" type="string" required>
  Comma-separated list of token addresses to track
</ParamField>

### Response

The endpoint returns a stream of Server-Sent Events:

**Initial Event:** Sent immediately upon connection with current balances

**Update Events:** Sent when balance or allowance changes are detected

**Ping Events:** Keep-alive messages sent every 20 seconds

### Event Types

**Balance Update Event:**
```
event: message
data: [{"balance":"5000000000000000000","allowance":"115792089237316195423570985008687907853269984665640564039457584007913129639935","token":{"address":"0xc02aaa39b223fe8d0a0e5c4f27ead9083c756cc2","decimals":18,"symbol":"WETH","name":"Wrapped Ether"}}]
```

**Keep-Alive Ping:**
```
event: ping
data: {}
```

### Headers

The SSE endpoint sets the following headers:

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
Access-Control-Allow-Origin: *
X-Accel-Buffering: no
```

## Notes

### Allowance Information

The allowance returned is specifically for the CoW Protocol vault relayer contract. This is the allowance checked when placing orders on CoW Swap.

A common max approval value is:
```
115792089237316195423570985008687907853269984665640564039457584007913129639935
```

### Balance Format

Balances are returned in the token's base unit (similar to wei for ETH):

- For tokens with 18 decimals: `1000000000000000000` = 1 token
- For tokens with 6 decimals (like USDC): `1000000` = 1 token

To convert to human-readable format:
```javascript
const humanReadable = balance / (10 ** decimals);
```

### SSE Connection Management

When using the SSE endpoint:

1. **Keep-alive pings** are sent every 20 seconds to maintain the connection
2. **Automatic cleanup**: When all clients disconnect, the server stops tracking that user
3. **Multiple clients**: Multiple clients can track the same user; the server aggregates tracked tokens
4. **Connection limit**: Be mindful of concurrent SSE connections

### Testing SSE Locally

You can test the SSE endpoint using the provided test page:

```bash
# Start the API server
yarn start

# Visit the test page
open http://localhost:3001/tests/balances
```

### Error Handling

| Status Code | Reason | Solution |
|------------|--------|----------|
| 400 | Invalid token addresses | Verify token addresses are valid Ethereum addresses |
| 400 | No tokens specified | Include at least one token in the query parameter |
| 500 | Database error | Check that the database is properly configured |

### Performance Considerations

- **Batch requests**: Query multiple tokens in a single request for better performance
- **SSE efficiency**: SSE connections only send updates when changes occur
- **Caching**: REST endpoint responses may be cached for a few seconds
